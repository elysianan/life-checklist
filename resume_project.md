# 人生已完成清单 - 简历项目描述

## 一句话介绍

一款帮助用户记录人生目标、可视化生命进度、留存成长回忆的移动端 Web App，包含清单管理、清单大全、人生轴、连续打卡、成就系统等 6 大模块，并通过「混合 AI」生成个性化人生报告。

---

## 项目描述（简历用）

**人生已完成清单** 是一款面向年轻人的**人生目标管理 + 时光记录工具**。产品以"人生进度"为核心概念，通过大环形进度条让用户直观感受时间流逝；通过"清单大全"提供 500+ 人生任务灵感；通过"人生轴"按时间线记录用户的成长轨迹；通过连续打卡、成就系统提升用户留存和活跃度。

项目采用 **HTML + Tailwind CSS + 原生 JavaScript** 实现，无需后端即可运行，数据通过 LocalStorage 本地持久化。项目已完成 6 大模块、15+ 核心功能，支持深色模式、移动端适配、动画交互、音效反馈。

最新版本引入 **AI 人生报告**：聚合用户完成数据，按「本月/本年/全部」生成个性化总结。采用「混合 AI」策略——默认本地规则引擎开箱即用，可选接入大模型 API（Kimi/DeepSeek/智谱），并在 API 超时/网络/CORS 异常时自动降级，兼顾体验、成本与可用性。

1. **差异化定位：** 不做传统 Todo List，而是做"人生时光记录仪"
2. **情感化设计：** 大环形人生进度条、每日格言、完成任务仪式感
3. **游戏化机制：** 连续打卡、成就徽章、等级系统、排行榜

6. **AI 赋能：** 混合 AI 策略（规则引擎 + 大模型）+ 优雅降级，把用户数据转化为个性化人生洞察

## 面试话术

### Q1：你为什么做这个项目？

**回答：**

我发现身边很多人都有很多"人生想做的事"，比如看一次极光、读完 100 本书、和爱的人做 100 件事，但这些目标往往散落在备忘录或脑海中，缺乏系统性的记录和追踪。所以我做了这个项目，帮助用户把人生目标可视化、把完成时刻记录下来，让生命更有仪式感。

### Q2：你在这个项目中解决了什么用户痛点？

**回答：**

主要解决了 4 个痛点：
1. **目标零散：** 人生愿望没有统一记录的地方
2. **不知道做什么：** 清单大全提供 500+ 灵感
3. **缺乏成就感：** 完成任务有动画、音效、成就徽章
4. **缺少成长回顾：** 人生轴按时间线记录每一次完成

### Q3：你怎么提升用户留存？

**回答：**

我设计了多层留存机制：
1. **连续打卡：** 利用损失厌恶心理，让用户不想断签
2. **成就系统：** 即时正向反馈，驱动用户完成更多任务
3. **人生轴：** 形成情感连接，用户会反复回看自己的成长
4. **照片记录：** 完成的不是任务，而是回忆
5. **每日格言：** 每天打开都有新的心灵激励

根据竞品数据，连续打卡可提升 7 日留存 25-35%，成就系统可提升 15-20%。

### Q4：如果继续迭代，你会做什么？

**回答：**

下一步我会重点做 3 件事：
1. **AI 赋能（已起步）：** 已完成 AI 人生报告；下一步做智能清单推荐、AI 每日格言、AI 目标拆解
2. **社交裂变：** 增加共同清单、挑战活动、排行榜，让用户邀请好友一起完成
3. **商业化：** 设计 Freemium 模式，会员解锁无限清单、云同步、PDF 人生报告

### Q5：你的技术选型是怎么考虑的？

**回答：**

我选择 HTML + Tailwind CSS + 原生 JavaScript，原因有 3 个：
1. **MVP 验证快：** 不需要后端和构建工具，2 天就能做出完整产品
2. **学习成本低：** 作为产品经理，原生 JS 更容易理解和修改
3. **可扩展性好：** 模块化代码结构，后续可以平滑迁移到 React/Vue

### Q6：聊聊你做的 AI 人生报告，有什么产品和技术上的思考？

**回答：**

这是我给产品加的第一个 AI 能力。**产品上**，我没有为了「AI」而 AI，而是抓住一个真实场景：用户积累了大量完成数据，却缺少一个回看、被「看见」的时刻。AI 报告把这些数据变成有温度的个性化总结，给用户一个每月回来的理由，也作为会员付费点。

**技术上**，我做了一个关键决策——混合 AI 策略：
- **默认本地规则引擎**：零配置、零成本、即开即用，保证每个用户都能用上；
- **可选真实大模型**：愿意配置 API Key 的用户能获得更自然的文案；
- **优雅降级**：真实 API 遇到超时、网络错误或浏览器 CORS 限制时，自动回退到规则引擎，保证「任何情况下都能出报告」。

这背后是产品经理要平衡的三件事：**用户体验、调用成本、接入门槛**。我还把 API Key 限制在本地存储、不经任何服务器，并配了 21 条单元测试保证核心逻辑可靠。
