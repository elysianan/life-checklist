# 人生已完成清单 v6.10.0 体验优化设计文档

**日期：** 2026-07-02  
**主题：** 修复 13 条用户反馈的交互与功能问题  
**版本：** v6.10.0  
**范围：** 我的页、首页、人生清单、任务详情、人生轴、分享与图片生成

---

## 一、背景与目标

v6.9.0 UX 优化完成后，用户在日常使用中反馈了 13 条具体的体验问题，涵盖交互缺失、Bug、功能不完善三类。本次优化目标是在**不引入新依赖、不大改架构**的前提下，把这 13 条问题全部修复并验证通过。

---

## 二、设计原则

1. **保持苹果极简风格**：白底圆角卡片、线型图标、蓝色强调色、统一阴影与圆角。
2. **局部修复为主**：在现有代码结构上直接修复，不借机做大规模重构。
3. **用户反馈即验收标准**：13 条逐条对应设计决策与验证用例。
4. **不写兼容代码**：数据迁移只做一次性补全，不保留旧分支逻辑。

## 三、问题清单与设计决策

### 3.1 「我的」页

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #1 「人生探索者」右侧箭头无功能 | 直接删除箭头，保持页面简洁 | `index.html`, `css/style.css` |
| #2 最近成就点击无反馈 | 已解锁成就点击跳转成就墙；未解锁时 toast 引导 | `js/profile.js` |

**详细设计：**
- 删除 `.profile-user-arrow` 容器及内部 SVG，保留头像、昵称、等级、进度文案。
- `#profile-achievements-preview` 绑定 `click`：
  - 若 `unlockedAchievements.length === 0`，显示 toast「还没有解锁成就，去完成几个任务吧」。
  - 否则调用 `showView('achievements')`。
- 点击时增加 `:active` 缩放反馈（`transform: scale(0.98)`）。

### 3.2 首页

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #3 今日格言无法自定义 | 点击格言卡片弹出编辑框，支持保存/恢复默认 | `index.html`, `js/app.js`, `js/storage.js`, `css/style.css` |
| #10 顶部统计建议删除「总任务」 | 只保留「总清单」和「已完成」两列 | `index.html`, `js/app.js`, `css/style.css` |

**详细设计：**
- 自定义格言：
  - 存储 key：`life_checklist_custom_quote`，结构 `{ text, author }`。
  - 首页加载时优先读取自定义格言；不存在时沿用现有随机格言池。
  - 模态框包含「格言内容」多行输入 + 「作者」单行输入 + 「保存」/「恢复默认」/「取消」。
  - 空内容不允许保存。
  - 自定义后固定显示该条，直到用户恢复默认或再次编辑。
- 统计简化：
  - `#home-view` 的 `.lists-overview` 移除「总任务」列及对应分隔线，改为两列均分。
  - `js/app.js` 中不再计算和渲染总任务数。
  - 废弃的 `#lists-view` 中同名统计也一并清理。

### 3.3 人生清单交互

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #4 清单记录点击后取消路径过长 | 短按 toggle 完成/未完成；长按仍打开任务详情 | `js/app.js` |
| #9 自定义模式下无法拖动已创建清单 | 修复删除角标对拖动事件的拦截 | `js/app.js`, `css/style.css` |
| #11 删除按钮与小圆点遮挡 | 删除角标移到卡片左侧外边，不占用内部空间 | `css/style.css`, `js/app.js` |

**详细设计：**
- 短按 toggle：
  - 已完成标签 → 未完成；未完成标签 → 已完成。
  - 同步更新首页卡片百分比、今日完成数、成就统计。
  - 视觉反馈：已完成用 `list.color` 填充白字，未完成用 `--border` 描边浅灰字。
- 长按仍然打开 `TaskDetailManager.showTaskDetail()`，保持 500ms 阈值与位移判定。
- 拖动排序修复：
  - 进入自定义模式后，删除角标只响应 click，不拦截卡片的 touch/mouse 事件。
  - 拖动排序持久化仍使用 `LIST_ORDER`。
- 删除角标布局：
  - 位置改为卡片左侧外边（负 margin 或绝对定位），与右上角小圆点/百分比完全分离。
  - 保持红色圆形 × 的苹果风格。

### 3.4 任务详情与照片

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #12 照片上传限制 2M 偏小 | 上限从 2MB 提升到 10MB | `js/taskDetail.js` |
| #13 添加大尺寸照片后无法拖动、按钮被遮挡 | 弹窗内容区可滚动 + 底部按钮固定 + 大图 lightbox | `css/style.css`, `js/taskDetail.js` |

**详细设计：**
- 照片上限：
  - 单张照片最大 10MB。
  - 超过时 toast「单张照片不能超过 10MB」。
  - 不做前端压缩，保持实现简单。
- 大图体验：
  - 任务详情弹窗内容区设置 `overflow-y: auto`，图片过高时可滚动。
  - 「删除任务」和「保存」按钮固定在弹窗底部。
  - 弹窗内图片默认 `max-height: 240px`，避免撑满首屏。
  - 点击图片打开全屏 lightbox，支持缩放与滚动查看。

### 3.5 人生轴

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #6 时间精度建议精确到日 | 事件对象扩展 `day` 字段，选择器改为年月日三列 | `js/datePicker.js`, `js/timeline.js`, `js/storage.js`, `test/timeline-test.cjs` |
| #7 双侧显示记录均在同一侧 | 修复 `_renderDouble`，按索引奇偶左右交错 | `js/timeline.js`, `css/style.css` |

**详细设计：**
- 数据模型：
  - 事件对象：`{ id, year, month, day, text }`。
  - 排序：先 `year`，再 `month`，最后 `day` 升序。
  - 显示格式：`2026年7月2日`。
  - 旧数据迁移：无 `day` 字段的事件补 `day: 1`，置 `timeline_migrated_v610` 标记避免重复。
- 选择器：
  - 添加/编辑弹窗从「年+月」双列扩展为「年+月+日」三列。
  - 复用 `DatePickerManager` 的三列日期能力。
  - 校验日期合法性（闰年、大小月）。
- 双侧布局：
  - `_renderDouble` 按事件索引奇偶分别渲染到右侧和左侧。
  - 偶数索引（0, 2, 4...）放左侧，奇数索引（1, 3, 5...）放右侧，或相反顺序，确保视觉上左右交错。
  - 左侧事件右对齐，右侧事件左对齐，中间保留中轴线。

### 3.6 生成图片与分享

| 反馈 | 设计决策 | 涉及文件 |
|---|---|---|
| #5 生成图片功能无法生成 | 修复 `html2canvas` 调用，增加错误处理 | `js/share.js`, `index.html` |
| #8 余生闹钟分享功能不完善 | 补全「保存图片」+「复制文案」+ 系统分享 | `js/lifeClock.js`, `js/share.js`, `css/style.css` |

**详细设计：**
- 清单详情生成图片：
  - 检查 `html2canvas` 是否正确加载。
  - 生成前确保目标元素可见（必要时临时移除 `hidden`）。
  - 用户上传图片若为本地 base64 则正常截图；若存在跨域资源则降级为只截文字区域。
  - 失败时 toast「图片生成失败，请重试」。
- 余生闹钟分享：
  - 点击「分享」弹出底部操作菜单：保存图片、复制文案、系统分享（若支持）。
  - 保存图片：截取 `#lifeclock-card` 生成 PNG 并触发下载。
  - 复制文案：生成如「我今年 26 岁，余生大约还有 27,000 天。认真活好每一天。」并写入剪贴板。
  - 系统分享：调用 `navigator.share({ files, title, text })`，不支持时隐藏该选项。

## 四、数据模型与存储变化

### 4.1 新增 / 变更的存储项

| Key | 类型 | 说明 |
|---|---|---|
| `life_checklist_custom_quote` | `{ text, author }` | 用户自定义格言 |
| `timeline_migrated_v610` | boolean | 人生轴 day 字段一次性迁移标记 |

### 4.2 数据结构变更

- 人生轴事件：
  ```js
  // 旧
  { id, year, month, text }
  // 新
  { id, year, month, day, text }
  ```
- 旧数据兼容：启动时检测无 `day` 字段的事件，自动补 `day: 1`。

### 4.3 删除的展示项

- 首页统计中的「总任务」列。
- 「我的」页「人生探索者」右侧箭头。

## 六、验证策略

### 6.1 单元测试

- 运行全部 `.cjs` 测试，确保无回归。
- 新增/补充：
  - `timeline-test.cjs`：`day` 字段排序、迁移、日期校验。
  - `datepicker-test.cjs`：三列选择器取值与高亮。
  - `share-test.cjs`：分享文案生成、html2canvas 失败降级。

### 6.2 手动验证清单

逐条对应 13 条反馈：

1. 「我的」页「人生探索者」右侧无箭头。
2. 已解锁成就点击跳转成就墙；未解锁 toast 提示。
3. 首页格言点击可编辑、保存、恢复默认。
4. 清单详情短按任务标签可完成/取消；长按仍开详情。
5. 清单详情「生成图片」成功导出 PNG。
6. 人生轴添加事件可选年月日，卡片显示完整日期。
7. 人生轴双侧布局左右交错。
8. 余生闹钟分享可保存图片、复制文案、系统分享。
9. 自定义模式下长按可拖动已创建清单。
10. 首页统计只显示「总清单」和「已完成」。
11. 自定义模式下删除角标不遮挡右上角内容。
12. 10MB 以内照片可正常上传。
13. 添加大尺寸照片后弹窗可滚动，底部按钮可见，点击大图可全屏查看。

### 6.3 视觉回归

- 首页、我的页、清单详情、人生轴（单列+双侧）、余生闹钟在浅色/深色主题下截图比对。
- 重点检查：删除角标位置、弹窗按钮、大图片溢出、双侧交错效果。

## 七、风险与应对

| 风险 | 应对 |
|---|---|
| 人生轴 day 字段迁移遗漏旧数据 | 启动时一次性扫描并补 `day: 1`，加迁移标记 |
| 清单标签 toggle 与长按手势冲突 | 保留现有计时器 + 位移阈值判定，仅把短按行为从「只能完成」改为 toggle |
| html2canvas 因用户上传大图跨域失败 | 本地图片通常为 base64，正常截图；失败时降级并 toast |
| 自定义格言与每日轮换逻辑冲突 | 自定义后固定显示，恢复默认后重新启用随机轮换 |
| 删除角标左移后小屏适配 | 使用负 margin 限制最大偏移，确保不超出视口 |
