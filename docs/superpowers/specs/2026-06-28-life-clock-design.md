# 余生闹钟（Life Clock）设计文档

- **日期**：2026-06-28
- **功能**：余生闹钟 —— 把首页的「人生倒计时」改造为「余生美好体验」，并独立成 tab 页
- **版本目标**：v6.5.0
- **状态**：设计待评审

---

## 1. 背景与动机

现有首页有一个「人生进度模块」：原生 `<input type="date">` 出生日期框、SVG 环形进度条（中央「人生进度 N%」）、「预期寿命 80」「已活天数」「剩余天数」等。

存在两个问题：

1. **交互不像手机端**：原生日期框是桌面端控件，手机上体验差。
2. **情绪是「倒计时」**：「人生已走过 N%」「剩余天数」本质是在提醒"时间在流失"，制造焦虑，观感不好。

本次改造的**核心理念**：把「人生倒计时」（焦虑）转化为「余生还能体验多少美好」（幸福、乐观、珍惜当下）。

---

## 2. 设计决策总览

| 维度 | 决策 |
|---|---|
| 形态 | 独立全屏页「余生闹钟」，底部导航**新增「余生」tab**（第 2 位） |
| 首页处理 | 移除原人生进度模块，改为一张**入口卡片**「你 26.56 岁了 →」，点击跳转「余生」tab |
| 时钟样式 | **iOS 风格模拟表盘**：黑色细圆环 + 黑色时针/分针 + 红色秒针 + 中心红点，无数字无刻度，秒针随真实时间走动 |
| 年龄显示 | 「你 **26.56079596** 岁了」，数字大号加粗，**末位实时跳动** |
| 余生呈现 | **2×2 网格**平铺 4 件美好事（默认：世界杯 / 夏天 / 退休 / 周末），可配置 |
| 寿命/退休假设 | 默认**寿命 100、退休 60**，仅用于计算、首页/页面不显示该数字；可在设置中调整 |
| 出生日期 | **手机端滑动滚轮选择器**（年/月/日三列），在余生页内触发；首次无生日时用随机日期演示 |
| 分享/保存 | 复用 `ShareManager` 的 html2canvas 截图能力，生成 PNG 分享/保存 |
| 整体风格 | 米白背景、极简、大留白 |
| 连续打卡 | 首页不再显示（底层逻辑与成就徽章保留，不动） |

## 3. 功能详述

### 3.1 余生闹钟页（独立 view `#lifeclock-view`）

页面自上而下结构（参考用户提供的设计图）：

```
┌───────────────────────────┐
│  ◀ 返回        余生闹钟     │  ← 顶部导航栏（返回 + 标题）
│                           │
│         ╭───────╮         │
│        │  ╲      │        │  ← iOS 风格表盘
│        │   ●──   │        │     黑时针/分针 + 红秒针 + 红中心点
│         ╲───────╱         │     按真实当前时间走动
│                           │
│     你 26.56079596 岁了    │  ← 年龄，数字大号加粗，末位跳动
│                           │
│         余生还可以          │  ← 灰色小标题
│   🏆 观看 14 届世界杯       │
│   🍦 享受 58 个夏天         │  ← 2×2 网格
│   🧍 还有 34 年退休         │
│   🌴 度过 3016 个周末       │
│                           │
│    [ ⬆ 分享 ]  [ 保存 ]    │  ← 两个按钮
│       设置退休年龄          │  ← 灰字链接
└───────────────────────────┘
```

**点击交互**：

- 点击**表盘或年龄区域** → 弹出滑动日期选择器，设置/修改出生日期。
- 点击「**设置退休年龄**」→ 弹出滚轮（或 stepper）设置退休年龄。
- 点击「**分享**」→ 生成图片并调用分享/下载。
- 点击「**保存**」→ 把整张余生闹钟卡片截图保存为 PNG。

### 3.2 表盘时钟

- **SVG 绘制**：一个圆环（`circle`，黑色 stroke）+ 三条指针（`line`）：时针（黑、短粗）、分针（黑、长）、秒针（红、细长）+ 中心圆点（红）。无刻度、无数字，保持极简。
- **走动**：根据**当前真实时间**（时、分、秒）计算指针角度并旋转。秒针每秒走动（用 `requestAnimationFrame` 平滑或每秒一跳）。
- **语义**：这是一个真实在走的钟，呼应"时间在流动"，但不带"还剩多少"的压迫感。

### 3.3 年龄实时跳动

- **计算**：`ageYears = (now - birthDate) / (365.25 × 24 × 3600 × 1000)`。
- **显示**：精确到小数点后 **8 位**，如 `26.56079596`。「你」「岁了」为较小的灰色字，中间数字大号加粗。
- **跳动**：用 `requestAnimationFrame` 持续刷新末几位，呈现飞速跳动的"时间流逝"感。
- **性能**：监听 `visibilitychange`，页面切到后台时暂停 tick，回到前台恢复，省电。

### 3.4 余生网格

- `余年 remainingYears = max(0, 寿命(100) - ageYears)`。
- 默认展示 4 件（参考图）：

| 图标 | 事件 | 公式 | 文案示例 |
|---|---|---|---|
| 🏆 | 世界杯 | `floor(余年 / 4)` | 观看 14 届世界杯 |
| 🍦 | 夏天 | `floor(余年)` | 享受 58 个夏天 |
| 🧍 | 退休 | `max(0, 退休60 - ageYears)` 取整 | 还有 34 年退休 |
| 🌴 | 周末 | `floor(余年 × 52)` | 度过 3016 个周末 |

> 上表「文案示例」的数字取自用户设计图（约 85 岁寿命）；实际按默认寿命 100 计算，余生数字会更大（更乐观），这正是把寿命设为 100 的用意。退休年数以「整岁」计算：`退休60 - floor(ageYears)`。

- **可配置候选池**（写入 `data.js` 的 `LIFE_EVENTS`，默认启用上面 4 个，后续可增删）：

| 图标 | 事件 | 公式 |
|---|---|---|
| 🌸 | 春天/花开 | `floor(余年)` |
| 🎆 | 春节 | `floor(余年)` |
| 🎂 | 生日 | `floor(余年)` |
| 🌕 | 满月 | `floor(余年 × 12.37)` |
| 🥇 | 奥运会 | `floor(余年 / 4)` |
| 🌅 | 日出 | `floor(余年 × 365.25)` |

- 图标默认用 emoji（零依赖、跨平台）；视觉打磨阶段可替换为 SVG 线性图标（更贴近设计图）。

### 3.5 底部导航新增「余生」tab

- 当前 5 个 tab：`🏠首页　📋进度　📚大全　📅人生轴　👤我的`。
- 新增后 6 个，「⏰余生」放第 2 位：`🏠首页　⏰余生　📋进度　📚大全　📅人生轴　👤我的`。
- 对应 `data-view="lifeclock"`，`showLifeClockPage()` 切换。

### 3.6 首页改造

- **移除**：整个「人生进度条模块」section（圆环、出生日期框、预期寿命卡、已完成/总任务卡）+ 「快速统计」section（已活天数、剩余天数、今日完成）+ 连续打卡显示。
- **新增**：一张**余生入口卡片**放在原位置，显示「你 26.56 岁了」+ 右侧箭头，点击调用 `showLifeClockPage()`。卡片年龄可静态显示（进入首页时计算一次，不必在首页跳动）。
- **保留**：今日格言、下方清单/AI 推荐区。

### 3.7 滑动日期选择器（`datePicker.js`）

- **UI**：底部上滑半屏卡片（遮罩 + 卡片），顶部「取消/确定」，三列滚轮：年 | 月 | 日。选中项居中放大高亮，上下虚化。
- **实现**：纯原生 JS + CSS `scroll-snap`（`scroll-snap-type: y mandatory`，每项 `scroll-snap-align: center`），监听 `scroll` 防抖取居中项。无第三方库。
- **数据**：年范围 1920 ~ 当前年；月 1-12；日按年月动态计算（闰年/大小月，28~31）；**最大不超过今天**（杜绝未来日期）。
- **确定**：写入 `birthDate`，刷新余生闹钟页。

### 3.8 随机占位（首次无生日）

- 首次进入余生页且无 `birthDate` 时，**随机生成一个演示生日**（1980-01-01 ~ 2005-12-31 间），让表盘/年龄/余生立刻"活"起来。
- 演示生日存入 `sessionStorage`（会话级，刷新页面不变、关闭浏览器重置），同一会话内固定不跳变；**不写入** localStorage 的正式 `BIRTH_DATE`。
- 顶部轻提示「👆 点这里设为你的生日」。用户正式选定后写入 `BIRTH_DATE`，演示态结束、提示消失。

### 3.9 寿命/退休设置

- **退休年龄**：余生页底部「设置退休年龄」链接 → 弹滚轮/stepper（范围 40~80，默认 60）。
- **预期寿命**：在「我的 - 设置」中新增一项可调（范围如 60~120，默认 100），与现有「AI 助手设置」并列。
- 修改后，余生闹钟页的余生数字重新计算。

## 4. 数据模型 / 存储

`storage.js` 的 `KEYS` 新增：

| Key | 常量名 | 默认值 | 说明 |
|---|---|---|---|
| `life_checklist_life_expectancy` | `LIFE_EXPECTANCY` | 100 | 预期寿命（岁），仅用于计算 |
| `life_checklist_retire_age` | `RETIRE_AGE` | 60 | 退休年龄（岁） |

- `BIRTH_DATE` 已存在，复用。
- 演示生日不进 localStorage，仅会话级。
- 新增存取方法：`getLifeExpectancy/setLifeExpectancy`、`getRetireAge/setRetireAge`（读取时空值回退默认）。

`data.js` 新增/修改：

- `DEFAULT_LIFE_EXPECTANCY`：80 → **100**。
- 新增 `DEFAULT_RETIRE_AGE = 60`。
- 新增 `LIFE_EVENTS` 配置数组（每项 `{ key, emoji, verb, unit, calc(remainingYears, ctx) }`）。

### 数据流

```
birthDate(或会话随机演示) + lifeExpectancy(100) + retireAge(60)
        │
        ▼
  LifeClockManager
   ├─ calcAge()           → 年龄(8位小数) ── rAF tick → 年龄区
   ├─ calcEvents()        → 余生 4 件事 ───────────→ 2×2 网格
   └─ renderClock(now)    → 真实时间指针 ── 每秒 ──→ SVG 表盘
```

## 6. 错误处理与边界（去焦虑兜底）

| 场景 | 处理 |
|---|---|
| 无生日 | 会话内随机演示生日 + 顶部引导提示 |
| 选择未来日期 | 选择器最大值限制为今天，无法选未来 |
| 寿命 ≤ 当前年龄（余年=0） | 余生网格显示正向兜底「每一天都是赚到 🎁」，不显示负数 |
| 退休年龄已过 | 「还有 N 年退休」换成「已自由 N 年 🌴」 |
| localStorage 读写异常 | try/catch 回退默认值 |
| html2canvas 加载失败 | toast 提示「图片生成失败，请重试」，页面不崩溃 |

## 7. 测试策略

`test/life-clock-test.cjs`（纯 Node，参考现有 `goal-breakdown-test.cjs` 风格，只测纯函数）：

- **年龄计算**：给定生日 + 固定 now，验证 `ageYears` 精度。
- **余生公式**：世界杯/夏天/退休/周末/满月等各事件数值正确。
- **边界**：生日为今天（age≈0）、寿命已超（余年=0 兜底）、退休已过、退休临界。
- **默认值**：寿命 100、退休 60。
- **存取**：set/get 寿命、退休（含空值回退默认）。

手动/浏览器验证：tab 切换、表盘走动、年龄跳动、滚轮选择、分享保存、首页入口卡片跳转。

7. 分享/保存能生成 PNG 图片。
8. 首页原人生进度模块已移除，显示余生入口卡片，点击跳转余生 tab。
9. 首页不再显示连续打卡（成就页打卡徽章保留）。
10. 单元测试全部通过。

## 9. 非目标（YAGNI）

- 不做云同步、跨设备。
- 不做提醒推送（纯前端受限）。
- 不引入新框架；html2canvas 沿用现有 CDN 动态加载方式，不新增本地依赖。
- 不改动连续打卡的底层逻辑与成就徽章（仅首页不显示）。
