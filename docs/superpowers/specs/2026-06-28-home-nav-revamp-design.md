- **日期**：2026-06-28
- **功能**：围绕「首页 = 我的人生清单」的整体重构 —— 导航 6→5 Tab、首页重做、清单详情网格标签、新增「人生进度」、人生轴改造为手动大事记、退休设置弹窗、AI 目标拆解移位
- **版本目标**：v6.6.0
- **状态**：设计待评审

## 1. 背景与动机

余生闹钟（v6.5.0）完成后，用户希望把应用的**首屏重心**从「人生进度/格言」转移到**清单本身**，并补齐几个体验缺口：

1. **首页没价值密度**：现在首页只有余生入口卡片 + 格言，用户想完成的「清单」却藏在第 3 个 Tab（进度）里，打开 App 看不到最该看的东西。
2. **余生 Tab 占位**：余生闹钟是"偶尔看一眼"的功能，不该常驻底部导航。
3. **清单条目交互弱**：城市/电影/歌曲这类清单，用户想"点一下名字就打卡"，现在要进详情、找勾选框，路径长。
4. **缺少多人物的人生进度视角**：用户想给自己、家人、朋友各建一条"人生进度条"。
5. **人生轴太被动**：现在只能"完成任务自动记录"，用户想**手动**记录人生大事（恋爱、分手、毕业…）并能分享。
6. **退休设置丑**：现在用浏览器原生 `prompt()`，与 App 风格割裂。

本次以用户提供的多张参考设计图为准，做一次较大的结构重构 + 两个新/改功能。

## 2. 目标与非目标

### 目标
- 首页变成「我的人生清单」，是打开 App 的第一落点。
- 底部导航精简为 5 个 Tab，余生改由首页入口进入。
- 清单详情改为网格标签，轻点打卡、长按看详情。
- 新增「人生进度」Tab（多人物）。
- 人生轴改造为手动大事记（单列/双侧/分享）。
- 退休设置改为 App 内弹窗。
- AI 目标拆解入口移到「清单」Tab 顶部。

### 非目标（YAGNI）
- 不做云同步 / 跨设备 / 账号体系。
- 不引入任何前端框架或第三方库（拖拽、滚轮均原生实现）。
- 不改动成就徽章、音效、AI 服务层等既有底层逻辑。
- 余生闹钟的表盘/年龄/网格视觉**保持现状**，本次只改"如何进入"和"退休怎么设"。

## 5. 各功能块详细设计

### 5.1 视图路由重构（技术地基，阶段 1）

**现状问题**：`app.js` 里每个 `showXxxPage()` 都手动 `classList.add/remove('hidden')` 切换 9~10 个视图，共 10 套近乎重复的代码。新增视图或改导航时极易漏改某个分支（上次会话已出现过"跨任务回归"）。

**方案**：引入统一路由。

```js
// 视图注册表：name → { el, nav, onShow, tick }
const VIEWS = {
  home:        { nav: 'home',        onShow: renderHomePage },
  templates:   { nav: 'templates',   onShow: () => TemplateManager.renderTemplateLibrary() },
  timeline:    { nav: 'timeline',    onShow: () => TimelineManager.render() },
  lifeprogress:{ nav: 'lifeprogress',onShow: () => LifeProgressManager.renderList() },
  profile:     { nav: 'profile',     onShow: () => ProfileManager.renderProfilePage() },
  detail:      { nav: 'home',        onShow: (id) => renderListDetail(id) }, // 二级页，导航高亮回 home
  lifeclock:   { nav: null,          onShow: () => LifeClockUI.show() },
  lifeprogressEdit: { nav: 'lifeprogress', onShow: (p) => LifeProgressManager.renderEdit(p) },
  // … report / achievements 等保留
};

function showView(name, arg) {
  Object.keys(VIEW_ELEMENTS).forEach(n => VIEW_ELEMENTS[n].classList.add('hidden'));
  VIEW_ELEMENTS[name].classList.remove('hidden');
  if (name !== 'lifeclock') LifeClockUI.stopTick();   // 离开余生页统一停 tick
  AppState.currentView = name;
  const v = VIEWS[name];
  if (v.nav) updateNavigation(v.nav);
  if (v.onShow) v.onShow(arg);
  window.scrollTo(0, 0);
}
```

旧的 `showHomePage/showListsPage/...` 改为 `showView('home')` 等的薄封装（保留函数名，减少调用点改动），或直接替换调用点。**收益**：新增"人生进度"等视图只需在注册表加一行；杜绝漏隐藏导致的"两个页面叠在一起"。

### 5.2 首页 = 我的人生清单（阶段 1）

**顶栏**（替换原 `home-view` header）：
- 左：⏰ 图标按钮 → `showView('lifeclock')`
- 中：标题「我的人生清单」
- 右：「自定义」文字按钮 → 进入**编辑模式**（清单卡片显示删除角标、长按可拖动；再点"完成"退出）。同时编辑模式下露出「+ 创建新清单」。

**今日格言**：保留一句（复用现有 `quote-card` + `updateDailyQuote()`），放标题下方。

**清单卡片**（参考图1，重做 `createListCard`）：
```
┌─────────────────────────────────┐
│ 中国的这些城市都去过哪些     0% │ ← 彩色标题条(list.color 背景)
├─────────────────────────────────┤
│ 北京 上海 广州 深圳 香港 澳门…  │ ← 浅色区，横排任务文字预览，溢出截断加…
└─────────────────────────────────┘
```
- 点卡片 → `showView('detail', list.id)`。
- 百分比来自 `StorageManager.calculateListProgress(list)`。
- 任务预览取前 N 个 `task.text`，用空格分隔，CSS 限制 2 行 + 省略号。

**长按拖动排序**：
- 长按某卡片 ≈500ms → 进入拖动态（卡片浮起、半透明）；移动 → 实时换位；松手 → 落位。
- 持久化：把排好序的 `list.id` 数组存入新 key `LIST_ORDER`；`renderListCards` 渲染时按此顺序（无记录则按 `lists` 原序）。
- 实现：监听 `touchstart/touchmove/touchend`（移动端）+ `mousedown/mousemove/mouseup`（桌面，便于本机测试）。手写，不引库。
- 底部提示「长按拖动排序」。
- **替代**现有的"排序方式菜单"（默认/进度/名称），自定义顺序即用户拖出来的顺序。

### 5.3 清单详情：网格标签（阶段 2）

**布局**（参考图2，重做 `detail-view`）：
```
< 返回                      编辑
┌─────────────────────────────────┐
│        中国的这些城市都去过哪些     │ ← 彩色标题条
│ ┌─────────────────────────────┐ │
│ │ 北京 上海 广州 深圳 香港 澳门 │ │ ← 标签网格(flex-wrap)
│ │ 台北 拉萨 南京 青岛 武汉 …    │ │   已完成标签=主题色填充
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
   主题配色      生成图片      移除清单   ← 底部操作栏
```
- **标签**：每个任务一个 `.task-tag`，未完成=浅灰描边，已完成=`list.color` 填充白字。
- **轻点**（tap）：切换该任务 completed → 复用现有 `handleTaskToggle` 的副作用（音效、撒花、首页卡片进度刷新、成就检查），但**就地更新标签样式**（不整页重渲染，避免闪烁）。
- **长按**（≈500ms）：打开 `TaskDetailManager.showTaskDetail(listId, taskId)`（现有弹窗：照片/笔记/优先级/完成日期/删除）。
- **顶部「编辑」**：进入标签编辑态——标签显示删除角标、末尾出现「+ 添加」标签（复用 `CustomManager.showAddTaskModal`）。
- **底部操作栏**：
  - 「主题配色」→ 颜色选择弹窗（复用 custom 的 color-picker），改 `list.color`。
  - 「生成图片」→ html2canvas 截图标签卡片为 PNG（复用 `ShareManager` 模式）。
  - 「移除清单」→ 删除确认（复用 `CustomManager.deleteList`），红色文字。
- **轻点 vs 长按判定**：`touchstart` 启动 500ms 计时器并记录坐标；`touchmove` 超阈值或 `touchend` 早于计时 → 视为 tap(打卡)；计时到 → 长按(详情)。避免误触。

### 5.4 人生进度（新功能，阶段 3）

**数据**：`persons = [{ id, name, birthDate, lifeExpectancy, color, isSelf }]`
- 始终存在一条 `isSelf: true`（称呼默认「我」），**不可删除**；其 `birthDate`/`lifeExpectancy` 与全局 `BIRTH_DATE`/`LIFE_EXPECTANCY`（余生闹钟用）**读写同一份**——即 `isSelf` 那条不单独存生日，而是代理到全局 key。改一处两处同步。
- 其他人物（家人/朋友）独立存于 `persons`。

**列表页**（参考图3）：
- 顶栏：标题「人生进度」+ 右上「+」→ `showView('lifeprogressEdit')`（新建）。
- 每条卡片：`称呼` ……右上 `已活岁/目标寿命`（如 `26/100`）；下方进度条（`person.color` 填充，宽度 = `floor(age)/lifeExpectancy`，条上居中显示 `26%`）。
- 点卡片 → 编辑该人物。
- `age = floor((now - birthDate)/MS_PER_YEAR)`，复用 `LifeClockEngine.calcAge`。无生日的人物显示「未设置生日」引导。

**添加/编辑页**（参考图2，全屏 `lifeprogress-edit`）：
- 顶栏：`< 返回`（不保存）/ 标题「添加」或「编辑」/ `保存`。
- 表单：
  - **称呼**：文本输入。
  - **出生日期**：**内联滚轮**（年/月/日三列，扩展 `DatePickerManager` 支持渲染到指定容器，而非弹出遮罩）。
  - **目标寿命**：数字输入（范围 40~150，默认 100）。
  - **主题色**：色块，点击展开颜色选择（复用 color-picker 8 色）。
- 保存：写入 `persons`；若 `isSelf` 则同步 `BIRTH_DATE`/`LIFE_EXPECTANCY`。
- 编辑非 isSelf 人物时，底部提供「删除」。

**与余生闹钟的一致性**：在余生闹钟改生日/寿命，回到人生进度，「我」那条同步更新；反之亦然。

### 5.5 人生轴改造（阶段 4）

**数据**：`events = [{ id, year, text }]`（按 `year` 升序展示）。

**迁移**（一次性）：首次加载新版本时，若检测到旧 `timeline` 结构（含 `taskId`/`date`），把每条转为 `{ id, year: date.getFullYear(), text: 原任务文字 }` 写入新 key，旧 key 清空；置 `timeline_migrated` 标记避免重复迁移。完成任务的代码路径**移除**自动写 timeline 的逻辑。

**列表页**（参考图3/4）：
- 顶栏：标题「人生时间轴」+ 右上分享图标。
- 布局切换段：`单列 / 双侧`（持久化到 `TIMELINE_LAYOUT`）。
  - **单列**：左列年份 + 圆点 + 竖线，右列事件卡片。
  - **双侧**：中轴圆点(内嵌年份) + 竖线，事件卡片左右交替（奇偶不同浅色底）。
- 空状态：居中插画 +「请点击加号添加人生事件」。
- 右下角绿色浮动「+」→ 添加事件弹窗（年份 + 描述文本）。
- 事件卡片点击 → 编辑/删除弹窗。

**分享卡**（参考图5）：
- 点右上分享 → 深色卡片：标题「我的人生时间轴」、副标题「共 N 件事 · 年份跨度 起—止」、按年份列出全部事件、底部署名「人生已完成清单 App · 记录每一段认真活过的日子」。
- 底部「保存」(红) / 「分享」(绿) / 关闭 ✕。复用 html2canvas。

### 5.6 退休设置弹窗（阶段 5）

- 替换 `lifeClock.js` 中 `prompt('设置退休年龄…')` 为 App 内 `modal-overlay` 弹窗：标题「设置退休年龄」+ 数字输入框(默认当前值) + 取消/确定。
- 校验：40~80 的整数，非法则抖动提示不关闭。
- 确定后 `StorageManager.setRetireAge(v)` 并刷新余生网格。

### 5.7 AI 目标拆解移位（阶段 5）

- 把 `goal-breakdown-entry` 入口卡片从 `lists-view`（已并入首页，将废弃）移到 `templates-view`（清单 Tab）顶部、分类导航之上。
- 调整 `GoalBreakdownUIManager` 绑定的容器/时机，确保在清单 Tab 渲染时入口可用。

## 9. 错误处理与边界

| 场景 | 处理 |
|---|---|
| 长按拖动中途松手/滚动 | 超时阈值与移动阈值区分"长按"与"滑动"，避免误触发拖动 |
| 轻点 vs 长按冲突 | 计时器 + 坐标位移阈值判定，二选一触发 |
| 人物无生日 | 进度卡显示「未设置生日」，进度按 0 处理，不报错 |
| isSelf 删除 | 禁止删除，编辑页不显示删除按钮 |
| 目标寿命 ≤ 当前年龄 | 进度封顶 100%，文案兜底（不显示 >100%） |
| 人生轴空 | 空状态插画 + 引导文案 |
| 人生轴年份非法/空 | 添加弹窗校验：年份为 1900~当前+预期范围的整数，描述非空 |
| 旧数据迁移异常 | try/catch，迁移失败则保留旧 key 并跳过，不阻断启动 |
| html2canvas 失败 | toast 提示，不崩溃 |
| localStorage 读写异常 | try/catch 回退默认 |

## 12. 风险与注意事项

- **路由重构是全局改动**：阶段1 必须把所有调用点（返回按钮、卡片点击、AI 入口等）切到 `showView`，逐一回归，防止漏改造成"点了没反应/两页叠加"。
- **跨阶段回归**：每阶段提交前，至少快速走查相邻功能（上次会话教训：子代理改共享 CSS 误删、跨任务回归）。CSS 改动尤其注意——新增类优先，谨慎删除旧类。
- **数据迁移只跑一次**：用 `TIMELINE_MIGRATED` 标记守卫，避免重复迁移把手动事件再次"翻译"。
- **isSelf 同源**：务必让「我」这条不持久化自己的生日，而是每次读写代理到全局 key，杜绝两份数据不一致。
