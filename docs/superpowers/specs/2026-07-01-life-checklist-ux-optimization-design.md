# 人生已完成清单 UX 优化设计文档

**日期：** 2026-07-01  
**主题：** 人生轴 / 人生进度 / 首页创建入口交互优化  
**版本：** v6.9.0（计划）

---

## 背景

v6.8.0 UI 重构完成后，用户在日常使用中发现三个核心痛点：

1. **人生轴添加事件**：年份滚轮选中态不明显，且同一年内无法区分多个月份的事件。
2. **人生进度添加人物**：出生日期内联滚轮缺少当前选中反馈。
3. **首页创建清单**：「创建新清单」藏在「自定义」编辑模式内，清单多时需要滑到底部才能操作。

本次优化聚焦以上问题，改动范围可控，不引入新依赖。

---

## 目标

- 人生轴事件支持**年月**级记录，年份滚轮以**当前年为起点倒序排列**。
- 所有滚轮选择器（年份、月份、日）都有清晰的**当前选中高亮**。
- 首页顶栏提供**一键创建新清单**入口。

---

## 设计方案

### 1. 人生轴：年月双列选择器

#### 1.1 弹窗选择器

将 `DatePickerManager.openYear()` 扩展为 `openYearMonth(currentYear, currentMonth, onConfirm)`：

- 弹窗内展示**年份 + 月份**两列滚轮。
- 年份列：`[2026, 2025, ..., 1920]`，默认停在当前年。
- 月份列：`[1, 2, ..., 12]`，默认停在当前月或传入值。
- 确认回调返回 `{year, month}`。

#### 1.2 数据模型

事件对象从：

```js
{ id, year, text }
```

扩展为：

```js
{ id, year, month, text }
```

- 新增 `month` 字段，取值 `1–12`。
- `TimelineEngine.migrate()` 对旧事件自动补 `month: 1`。
- 排序规则改为：先按 `year` 升序，同一年内按 `month` 升序。

#### 1.3 显示格式

时间轴卡片统一显示为：

```
2026年1月
```

此格式在中文语境下最自然，与「人生大事记」的场景气质匹配。

#### 1.4 旧数据迁移说明

已有事件只保存了年份。打开 App 时，兼容逻辑会自动给这些事件补上 `month: 1`，表示「已知年份、月份待补」。用户后续可点击事件进入编辑，改为真实月份。

---

### 2. 滚轮当前选中视觉反馈

#### 2.1 统一高亮机制

在 `DatePickerManager` 中新增 `_updateActiveItem(ul)`：

- 根据 `ul.scrollTop` 计算中心索引。
- 给中心 `<li>` 添加 `.dp-item-active`，移除其他项的该类。
- 绑定到所有滚轮的 `onscroll` 事件（popup 与内联模式均生效）。

#### 2.2 视觉样式

```css
.dp-item-active {
  color: var(--accent);
  font-weight: 600;
  transform: scale(1.08);
  transition: all 0.15s ease;
}
```

高亮项使用主色、加粗、轻微放大，让用户一眼识别当前选中年/月/日。

---

### 3. 人生进度生日选择器反馈

人生进度编辑页使用 `DatePickerManager.renderInline()` 渲染内联三列滚轮。复用第 2 节的 `_updateActiveItem`，让年/月/日三列同时高亮中心项，解决「不知道当前选的是哪一天」的问题。

此改动只涉及交互反馈，不改动数据模型。

---

### 4. 首页快速创建按钮

#### 4.1 布局调整

首页顶栏从：

```
[⏰]        [自定义]
```

调整为：

```
[⏰]        [+] [自定义]
```

- 左侧保留「余生闹钟」入口。
- 右侧新增圆形「+」按钮，直接调用 `CustomManager.showAddListModal()`。
- 「自定义」按钮继续负责切换编辑/完成模式。

#### 4.2 行为

- 「+」按钮**始终可见**，不受编辑模式影响。
- 进入编辑模式后，原底部「创建新清单」卡片仍然保留，方便拖拽排序后顺手添加。
- 点击「+」弹出与原来一致的「创建新清单」模态框。

## 测试策略

- **单元测试**：
  - `TimelineEngine.sortByYear()` 改为按 `year` + `month` 排序。
  - `TimelineEngine.migrate()` 验证旧数据补 `month: 1`。
  - `DatePickerManager._yearRange()` 验证倒序且包含当前年。
- **手动验证**：
  - 人生轴添加事件：选择 2026 年 3 月，卡片显示「2026年3月」。
  - 人生轴同一年添加 1 月、2 月、3 月事件，确认排序正确。
  - 人生进度添加人物：滚动生日选择器，确认中心项高亮。
  - 首页点击「+」直接弹出创建模态框。
